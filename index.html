영화 이미지 다운로드

• 문제: 서버에서 이미지 URL을 받고 다시 그 이미지 URL을 파싱해서 이미지를 가져오는 부분이 잘 풀리지 않았습니다.

• 해결 방안: 이미지 데이터를 관리하는 매니저 클래스를 따로 만들어서 해결했습니다. 셀을 로딩하는 시점에는 이미지 URL 정보를 알고 있으므로 데이터를 먼저 가져온 다음에 셀을 로딩하는 시점에 백그라운드에서 이미지를 다운로드하도록 구현했습니다.

• 관련 코드: https://wiggly-subway-73d.notion.site/1e10298ba8b64a08bc0b0e19b1f35107

테이블뷰 안에 있는 컬렉션뷰의 인덱스를 메인 화면에 가져오기

• 문제: 테이블뷰 안에 있는 컬렉션뷰의 인덱스를 메인 화면에 가져와 해당 인덱스에 맞는 영화 이미지를 네트워크 통신을 통해 가져와야 하는데 올바른 인덱스를 가져오기가 어려웠습니다.

• 해결 방안: 프로토콜을 사용하여 문제를 해결했습니다. 구글링하다가 프로토콜이 상속으로 멀리 떨어져 있는 두 클래스가 특정 목표를 달성하기 위해 서로 통신할 수 있도록 한다는 걸 알게 되었습니다. 그래서 CollectionViewCellDelegate 프로토콜을 만들어서 테이블뷰에 속성으로 선언한 후 메인 화면 뷰컨트롤러를 대리자로 지정했습니다. 그 후 메인 화면 뷰컨트롤러에서 해당하는 작업을 처리했습니다.

• 관련 코드: https://wiggly-subway-73d.notion.site/925518b935834f7b820174a65731c13e

백그라운드 화면에 Dim View 넣기

• 문제: 리뷰 작성 화면을 띄울 때 백그라운드 화면에 Dim View를 추가하고 화면이 내려가면 Dim View를 삭제하는 기능을 구현하기가 어려웠습니다.

• 해결 방안: window에 Dim View를 추가하는 방법으로 해결했습니다. 구글링 결과, Dim View를 추가할 때, window 객체가 있는지 확인하고 있다면 addSuview 메소드를 이용하여 window에 Dim View를 추가한다는 걸 알게 되었습니다. 그래서 리뷰 화면을 띄우기 전에 이전 화면의 prepare 메소드에서 window에 Dim View를 추가하여 백그라운드 화면에 Dim View를 넣었습니다. 리뷰 화면이 닫힐 때는 close 메소드에서 notification을 보내고 이 notification을 이전 화면에서 받아 Dim Viwe를 super view에서 제거하도록 구현했습니다.

• 관련 코드: https://wiggly-subway-73d.notion.site/Dim-View-00ed7720972a4ed0ae188df1d24a9078

가게 정보 필터링 기능

• 문제: 테이블뷰 셀에 선택된 필터링 항목을 뷰컨트롤러에 가져오는 과정에서 셀에 있는 데이터가 뷰컨트롤러로 잘 전달되지 않았습니다.

• 해결 방안: cellForRow(at:) 메소드를 이용해 해당 IndexPath에 있는 테이블뷰 셀을 가져온 후 셀에 접근해서 데이터를 가져오는 것으로 해결했습니다. 셀에 직접 접근해서 처리하는 것은 마지막 수단으로 남겨둬야 한다고 배웠습니다. 하지만 notification, 델리게이트 패턴 등 여러 가지 방법을 시도해본 결과, 해결되지 않았기 때문에 직접 접근하는 방법을 사용했습니다.

• 관련 코드: https://wiggly-subway-73d.notion.site/3c6ed33f38384ed2ae9567a326a433ad

필터링 항목의 선택 상태에 따른 색상 변경

• 문제: 필터링할 항목의 선택 상태에 따라 배경색을 변경되도록 구현하려고 하는데 어떤 걸 기준으로 선택 상태를 구분할지 고민했습니다.

• 해결 방안: 간단하게 생각해서 필터링 항목의 현재 색상을 기준으로 구분했습니다. 현재 필터링 항목의 UI를 감싸고 있는 컨테이너뷰의 색상을 기준으로 색상이 검정이면 선택되지 않은 상태, 빨강이면 선택된 상태로 구분했습니다.

• 관련 코드: https://wiggly-subway-73d.notion.site/6cac505e50614160b3066aa7c314331a
